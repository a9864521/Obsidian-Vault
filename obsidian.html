1. Clone repository...（複製專案）

用途：把「別人的」或「你在 GitHub 網站上的」專案下載到電腦

2. Create new repository...（建立新專案）⭐

用途：從零開始建立一個全新的專案

3. Add existing repository...（加入現有專案）

用途：把「電腦上已經存在的資料夾」加入 Git 管理

1. Name（專案名稱）⭐ 必填

  • 輸入你的專案名稱
  • 資料夾名稱會成為 GitHub 網址

2. Description（專案描述） 選填

  • 簡單說明這個專案是什麼

3. Local path（本機路徑）⭐ 重要

  • 專案要儲存在電腦的哪裡

4. Initialize this repository with a README

  • ☑️ 建議勾選
  • 會自動建立一個 README.md 檔案（專案說明文件）
  • 方便之後寫專案介紹

5. Git ignore

  • 告訴 Git 哪些檔案「不要」上傳到 GitHub
  • 避免上傳不必要的大型檔案（如套件資料夾）
  • 避免上傳敏感資訊（如密碼、API 金鑰）

6. License

  • 告訴別人可以如何使用你的程式碼
  • MIT License：最寬鬆，別人可以自由使用、修改、商用
  • Apache License 2.0：類似 MIT，但有專利保護
  • GNU GPL：別人修改後也必須開源
  • None：如果是私人專案或不想讓人使用

提交變更

  • Summary（必填）：簡短描述這次改了什麼（例如：「新增登入功能」）
  • Description（選填）：詳細說明

推送選項

  • Open in Visual Studio Code"（用 VSCode 開啟）
    • 快速開啟這個專案資料夾到 VSCode
    • 方便你開始編輯檔案
  • "Show in Explorer"（在檔案總管中顯示）
    • 打開專案資料夾的位置
    • 如果你想改路徑，就點這個看資料夾在哪

設定選項

  • Name（專案名稱）
  • Description（專案描述）
  • Keep this code private（保持程式碼私密）
